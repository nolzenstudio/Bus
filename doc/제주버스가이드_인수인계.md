작성일: 2026-07-22 · 최종 수정: 2026-07-22 (배포 및 여정 선택 흐름 반영)

## 1. 프로젝트 개요

**목표:** 사용자의 현재 위치(GPS)를 기준으로 가장 가까운 버스 정류장을 추천하고, "이 정류장에서 특정 버스를 타고 어디까지 갈 수 있는지"와 그 버스의 실시간 도착예정시간을 보여주는 모바일 반응형 웹앱.

**핵심 흐름 (여정 선택, 2단 중첩 아코디언):**
1. 브라우저에서 GPS 위치 권한 요청 → 내장 정류장 데이터(4,079곳)에서 가장 가까운 8곳 즉시 계산
2. **정류장 선택** → 그 카드 바로 아래로 '지나는 버스' 목록 확장 (재클릭 시 접힘)
3. **버스(노선) 선택** → 그 아래로 '도착 정류장' 목록 확장 — 갈 수 있는 정류장을 거리순 표시 (재클릭 시 접힘)
4. **도착 정류장 선택** → 실시간 도착 전광판에 출발→도착 요약 + 그 버스의 도착예정정보만 필터링해 LED 스타일로 표시, 20초마다 자동 갱신

**기술 스택:** 순수 HTML/CSS/JS (프레임워크 없음, 단일 파일), Cloudflare Workers (CORS 프록시)

## 2. 아키텍처

```
[사용자 브라우저]
      │  GPS 좌표 확인 + 내장 정류장 데이터로 최단거리 계산 (API 호출 없음)
      │
      │  정류장 선택 시 fetch()
      ▼
[Cloudflare Workers 프록시]  https://jeju-bus-proxy.nolzenstudio.workers.dev
      │  (서버 간 통신이라 CORS 제약 없음)
      ▼
[bus.jeju.go.kr]  실시간 도착예정정보 등 제공
```

### 왜 프록시가 필요한가
`bus.jeju.go.kr`의 API는 자체 홈페이지 화면에서만 호출하도록 만들어져 있어, 다른 도메인(우리 웹앱)에서 브라우저로 직접 `fetch()`하면 **CORS 정책에 의해 차단**됩니다 (`Access-Control-Allow-Origin` 헤더 없음, 실측 확인됨). 이를 우회하기 위해 Cloudflare Workers를 중간 서버로 두어, 서버-서버 통신으로 대신 요청하고 결과에 CORS 허용 헤더를 붙여 돌려줍니다.

## 4. Cloudflare Workers 프록시

- **배포 주소:** `https://jeju-bus-proxy.nolzenstudio.workers.dev`
- **Cloudflare 계정:** nolzenstudio@gmail.com
- **Worker 이름:** `jeju-bus-proxy`

### 현재 배포된 코드 (worker.js)

```javascript
export default {
  async fetch(request, env) {
    if (request.method === "OPTIONS") {
      return new Response(null, { headers: corsHeaders() });
    }

    const url = new URL(request.url);
    let targetUrl;

    if (url.pathname.startsWith("/jejudata/")) {
      // 예: /jejudata/station?stationName=한국병원
      //  -> open.jejudatahub.net/api/proxy/{데이터셋ID}/{appkey}?stationName=한국병원
      const datasetId = "DD11ab6a6t11D16baaa1a2tD26ata161";
      targetUrl = `https://open.jejudatahub.net/api/proxy/${datasetId}/${env.JEJU_DATAHUB_KEY}${url.search}`;
    } else {
      // 예: /searchArrivalInfoList.do?station_id=405000155
      //  -> bus.jeju.go.kr/api/searchArrivalInfoList.do?station_id=405000155
      targetUrl = "https://bus.jeju.go.kr/api" + url.pathname + url.search;
    }

    const upstream = await fetch(targetUrl, {
      method: "GET",
      headers: { "Referer": "https://bus.jeju.go.kr/" }
    });

    const body = await upstream.text();

    return new Response(body, {
      status: upstream.status,
      headers: {
        "content-type": "application/json;charset=UTF-8",
        ...corsHeaders()
      }
    });
  }
};

function corsHeaders() {
  return {
    "Access-Control-Allow-Origin": "*",
    "Access-Control-Allow-Methods": "GET, OPTIONS",
    "Access-Control-Allow-Headers": "*"
  };
}
```

### 환경 변수 (Secrets)
Worker의 **Settings → Variables and Secrets**에 등록됨:

| 이름 | 용도 | 현재 사용 여부 |
|---|---|---|
| `JEJU_DATAHUB_KEY` | 제주데이터허브(jejudatahub.net) appkey | 현재 최종본(`-final.html`)에서는 미사용. 정류장 좌표를 CSV 내장 방식으로 대체했기 때문. 추후 정류장 데이터 최신화 시 재사용 가능 |

### 호출 방식
프록시 주소 뒤에 `bus.jeju.go.kr/api/` 이하 경로를 그대로 붙이면 됩니다. 예:
```
https://jeju-bus-proxy.nolzenstudio.workers.dev/searchArrivalInfoList.do?station_id=405002676
```

## 5. 사용 중인 API 명세

모두 `bus.jeju.go.kr`이 원출처이며, 위 프록시를 경유해 호출합니다. 별도 인증키 불필요 (제주특별자치도 자체 공개 API).

### 5-1. 노선 목록 — `searchBusRouteList.do`
전체 버스 노선을 한 번에 반환 (파라미터 없음, 화면에서 필터링하는 방식으로 추정).

```json
{
  "ROUTE_ID": 405410111,
  "ROUTE_NUM": "101",
  "ROUTE_NM": "101-11",
  "ROUTE_SUB_NM": "남원환승정류장(남원읍사무소)",
  "ORGT_NM": "제주국제공항2(일주동로, 516도로)",
  "ORGT_STATION_ID": 405002101,
  "DST_NM": "남원환승정류장(남원읍사무소)",
  "DST_STATION_ID": 406001604,
  "MIN_INTERVAL": 0,
  "MAX_INTERVAL": 0
}
```

### 5-2. 정류장 통과 노선 — `searchRouteByStationList.do?station_id={ID}` ⭐ 핵심
특정 정류장을 지나가는 노선 목록. **여정 선택 2단계('지나는 버스' 목록)에서 사용 중.** 각 항목의 `ROUTE_ID`는 **방향까지 구분된** 값이라, 같은 번호라도 상·하행이 서로 다른 `ROUTE_ID`로 나옵니다(예: 900-1 `405090001`→조천리, 900-2 `405090002`→반대). 앱은 이를 `ROUTE_NUM + DST_NM`(번호+방향)으로 묶어 리스트에 표시합니다.

```json
{
  "STATION_ID": 405002676,
  "STATION_NM": "김녕농협하나로마트",
  "ROUTE_ID": 405090001,
  "ROUTE_NUM": "900",
  "ROUTE_NM": "900-1",
  "ORGT_NM": "김녕장례문화센터",
  "DST_NM": "조천리",
  "DST_STATION_ID": 405000234
}
```

### 5-3. 정류장 도착예정정보 — `searchArrivalInfoList.do?station_id={ID}` ⭐ 핵심
현재 앱에서 실시간 도착시간 표시에 사용 중인 API.

```json
{
  "ARRV_STATION_ID": 405000974,
  "ROUTE_ID": 405320111,
  "ROUTE_NUM": "201",
  "ROUTE_SUB_NM": "서귀포버스터미널",
  "PLATE_NO": "제주79아3226",
  "VH_ID": 7983226,
  "PREDICT_TRAV_TM": 3,
  "REMAIN_STATION": 2,
  "CURR_STATION_NM": "평대리동동",
  "CURR_STATION_ID": 405001315,
  "LOW_PLATE_TP": "N"
}
```
- `PREDICT_TRAV_TM`: 도착까지 남은 시간(분)
- `REMAIN_STATION`: 남은 정류장 수
- **버스가 없거나 배차 간격이 길면 빈 배열 `[]`을 반환함** (정상 동작, 앱에서 "지금은 접근 중인 버스가 없어요"로 처리 중)

### 5-4. 노선경유정류장 조회 — `searchBusRouteStationList.do?route_id={ID}` ⭐ 핵심
한 노선(계통)이 순서대로 지나가는 정류장 목록. **여정 선택 3단계('도착 정류장' 목록)에서 사용 중.** 선택한 노선 그룹의 모든 계통 `ROUTE_ID`에 대해 병렬 호출한 뒤, 출발 정류장의 `STATION_ORD` 이후 정류장만 모아 `ROUTE_DIST` 차이로 거리를 계산하고 병합해 거리순으로 보여줍니다.

```json
{
  "ROUTE_ID": 405090001,
  "ROUTE_NUM": "900",
  "STATION_ORD": 2,
  "STATION_ID": 405002676,
  "STATION_NM": "김녕농협하나로마트",
  "ROUTE_DIST": 888
}
```
- `STATION_ORD`: 기점부터 몇 번째 정류장인지
- `ROUTE_DIST`: 기점부터의 누적 거리(m)

### 5-5. 전체 운행버스 위치 — `searchBusAllocList.do`
현재 운행 중인 모든 버스의 실시간 GPS 좌표. 파라미터 없이 전체 반환.

```json
{
  "VH_ID": 7983064,
  "PLATE_NO": "제주79아3064",
  "ROUTE_ID": 405410111,
  "ROUTE_NUM": "101",
  "LOCAL_X": 126.861152,
  "LOCAL_Y": 33.518313,
  "CURR_STATION_ID": 405000974,
  "CURR_STATION_NM": "세화환승정류장(세화리)",
  "LOW_PLATE_TP": "N"
}
```
- `LOCAL_X` = 경도, `LOCAL_Y` = 위도
- 초기 버전(`jeju-bus-guide-live.html`)에서는 이 데이터로 "현재 운행 중인 정류장" 목록을 만들었으나, 최종본에서는 CSV 내장 방식으로 대체됨. **지도에 버스 아이콘을 실시간으로 표시하는 기능을 추가할 때 재사용 가능**.

### 5-6. (미사용) 제주데이터허브 정류소 검색
```
https://jeju-bus-proxy.nolzenstudio.workers.dev/jejudata/station?stationName={정류장명}
```
정류소 이름으로 좌표를 검색하는 API (jejudatahub.net 제공). 이름 검색만 가능하고 좌표/반경 검색은 미지원 확인됨. 현재 최종본에서는 CSV 내장 방식으로 대체되어 미사용이지만, appkey는 살아있으므로 필요 시 재사용 가능.

## 6. 정류장 데이터 (CSV 내장)

- **원본 파일:** `제주특별자치도_버스정류소현황_20221101.csv` (사용자 제공, 공공데이터)
- **기준일자:** 2022-11-01 — **이후 신설/폐지된 정류장은 반영 안 됨** (알려진 한계점)
- **인코딩:** CP949 (EUC-KR 계열) → UTF-8로 변환 후 내장
- **원본 컬럼:** 정류소아이디, 정류소명, 경도, 위도, 위치정보(주변설명), 데이터기준일자
- **내장 형식:** `[정류소ID, 정류소명, 경도, 위도]` 튜플 배열로 압축, `jeju-bus-guide-final.html` 안에 `STATIONS` 상수로 직접 포함 (총 4,079건, 약 170KB)
- **매핑 확인됨:** 이 CSV의 `정류소아이디`가 `bus.jeju.go.kr` API들의 `station_id`/`STATION_ID`와 동일한 체계임을 실제 호출로 확인함 (예: `405002676` = 김녕농협하나로마트)

### 최단거리 계산 로직 (하버사인 공식)
```javascript
function haversine(lat1, lng1, lat2, lng2){
  const R = 6371000;
  const toRad = d => d * Math.PI / 180;
  const dLat = toRad(lat2-lat1);
  const dLng = toRad(lng2-lng1);
  const a = Math.sin(dLat/2)**2 + Math.cos(toRad(lat1))*Math.cos(toRad(lat2))*Math.sin(dLng/2)**2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
}
```
GPS 좌표를 받으면 4,079개 정류장 전체와의 거리를 즉시 계산(API 호출 없음) → 오름차순 정렬 → 상위 8곳 표시.

## 8. 진행 경과 요약 (트러블슈팅 기록)

1. **초기 요구사항:** 위치 기반 정류장 추천 + 실시간 도착정보 → 목업 데모(`jeju-bus-guide.html`)로 UI/UX 먼저 확정
2. **실 API 조사:** `bus.jeju.go.kr`의 내부 API 엔드포인트들을 사용자가 브라우저 개발자도구(F12 Network 탭)로 직접 캡처하여 확보
3. **CORS 문제 발견:** `jeju-bus-api-test.html`로 직접 호출 테스트 → 응답 헤더에 `Access-Control-Allow-Origin` 없음 확인 → 브라우저 직접 호출 불가 판명
4. **프록시 구축:** Cloudflare Workers로 CORS 우회 프록시 배포, 여러 차례 배포 이슈 해결 (Error 1031은 미리보기 창 문제로 무관, "Apply" 버튼으로 실제 배포 적용 필요했음)
5. **API 필드 구조 확인:** 노선 목록 → 정류장 통과 노선 → 도착예정정보 → 노선경유정류장 → 전체 버스 위치 순으로 실제 호출하며 필드명 확인
6. **정류장 검색 방식 결정:** 처음엔 jejudatahub 이름 검색 시도 → 반경 검색 미지원 확인 → "노선별(현재 운행 중) 정류장"으로 임시 구현(`-live.html`) → 사용자가 공식 정류소 CSV(4,079건)를 제공하며 전체 데이터 내장 방식으로 최종 업그레이드(`-final.html`)

## 12. 여정 선택 흐름 & UI 구조 (초판 이후 추가)

초판의 "정류장 선택 → 그 정류장의 모든 도착정보 표시"에서, **"이 정류장에서 특정 버스를 타고 어디까지 갈 수 있는지"**까지 안내하는 여정 선택 흐름으로 확장했습니다.

### 12-1. 4단계 흐름 (2단 중첩 아코디언)

```
가까운 정류장(8곳)
  └─[정류장 클릭·확장]─ 지나는 버스 (번호+방향으로 그룹화)
        └─[노선 클릭·확장]─ 도착 정류장 (계통 병합, 거리순)
              └─[도착지 클릭]─ 실시간 전광판 (출발→도착 요약 + 그 버스만 필터)
```

- **확장/축소 토글:** 정류장·노선 카드를 다시 클릭하면 접힘.
- **DOM 이동 방식:** 상세 패널(`#stationDetail`, `#routeDetail`)을 선택 카드 아래 슬롯(`.station-slot`, `.detail-slot`)으로 `appendChild` 이동하고, 리스트를 다시 그리기 전엔 보관함(`#stationDetailHolder`, `#detailHolder`)으로 회수한다. 이렇게 하면 `timelineArea`/`boardArea` 등 고정 ID를 유지한 채 렌더 함수를 재사용할 수 있고, 재렌더 시 노드가 파괴되지 않는다.

### 12-2. 노선 그룹화 (핵심 로직)

제주 버스는 **한 노선번호 아래 수십 개의 세부 계통(운행 변형)**이 있어, 허브 정류장에서 같은 번호가 여러 번 나온다(예: 제주버스터미널의 `232`가 12개, `201`이 9개). 각 계통은 서로 다른 `ROUTE_ID`를 가진 별개 노선이다(완전 중복 `ROUTE_ID`는 없음).

- **리스트:** `ROUTE_NUM + DST_NM`(번호+방향)으로 묶어 1줄로 표시(예: "232 · 제주버스터미널 방면 · 9개 계통 운행").
- **도착지 목록:** 그룹의 모든 계통 `ROUTE_ID`에 `searchBusRouteStationList.do`를 병렬 호출 → 출발 정류장(`STATION_ORD`) 이후 정류장만 모아 병합. 정류장별로 최소 거리(`ROUTE_DIST` 차)와 '지나는 계통 집합'을 기록하고 거리순 정렬. 일부 계통만 지나는 정류장엔 `n/N 계통` 배지.
- **전광판 필터:** `searchArrivalInfoList.do` 결과를 **선택한 도착지를 실제로 지나는 계통 `ROUTE_ID` 집합**으로만 필터. → 같은 번호라도 그 도착지를 안 가는 계통·반대 방향은 제외하되, 먼저 오는 다른 계통은 놓치지 않음.

### 12-3. 위치 권한 UX

- **권한 거부(`err.code === 1`) 감지:** iOS(Safari)/안드로이드(Chrome) 재설정 안내 카드 표시. `navigator.permissions`로 시작 시 상태를 선확인해 이미 거부면 프롬프트 없이 안내부터 노출. `userAgent`로 OS 자동 감지.
- **제주 밖 위치:** 가장 가까운 정류장이 40km(`OUT_OF_RANGE_M`) 초과면 범위밖 배너 표시(목록은 참고용으로 유지).

### 12-4. 주요 함수 맵 (index.html `<script>`)

| 함수 | 역할 |
|---|---|
| `initLocate` / `startLocate` | 권한 선확인 → GPS 위치 획득(실패 시 안내/기본위치) |
| `loadNearbyStations` / `renderStations` | 4,079곳 하버사인 거리 → 가까운 8곳(+범위밖 배너) |
| `selectStation` | 정류장 선택/토글 → `mountStationDetail` → `loadRoutes` |
| `loadRoutes` / `renderRoutes` | 통과 노선 조회 → 번호+방향 그룹화 리스트 |
| `selectGroup` | 노선 그룹 선택/토글 → `mountDetail` → `loadGroupTimeline` |
| `loadGroupTimeline` / `renderGroupTimeline` | 계통 병렬조회·병합 → 도착 가능 정류장 거리순 |
| `selectGroupDest` | 도착지 선택 → 계통 집합 확정 → `loadArrivals` |
| `loadArrivals` / `renderBoard` | 도착예정 조회 → 계통 필터 → 전광판(20초 갱신) |
| `mount*/park*Detail`, `reset*` | 상세 패널 DOM 이동·상태 초기화 |
